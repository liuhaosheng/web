http 协议的 主要特点

简单快捷 / 灵活 / 无链接 / 无状态

http 报文的组成部分

请求报文 请求行 - 请求头 - 空行 - 请求体

  • 请求行包括：http方法，页面地址，http协议，版本
  • 请求头： key value 值
  • 空行： 区分请求头和请求体的

相应报文 状态行 - 相应头 - 空行 - 相应体

  • 状态行: http协议，版本，状态码，
  • 相应头 对应请求头

post 和 get 的区别

GET的语义是请求获取指定的资源。GET方法是安全、幂等、可缓存的

POST的语义是根据请求负荷（报文主体）对指定的资源做出处理，POST不安全，不幂等，（大部分实现）不可缓存。

为什么 post 会比 get 传的数据量大？

get 不支持 二进制和 url 的长度限制

  • get 在浏览器回退时是无害的，而post 会再次提交请求
  • get 产生的 url 地址可以被收藏，而 post 不会
  • get 请求会被浏览器默认缓存， 而 post 不会，除非手动设置
  • get 请求在 url 中传送参数有长度限制，而post 没有
  • get 只能进行 url 编码， 而 post 支持多种编码方式
  • get 只接受 ASCII 字符，而 post 没有限制
  • get 请求参数会保存在浏览器种，而 post 种的参数不会
  • get 参数通过 url 传递， post 放在 Rrequest body

http 状态码

  • 1XX ：指示信息 - 指示请求已接受，继承处理
  • 2XX ：成功 - 表示请求已被成功接受
    • 200: 请求成功
  • 3XX ：重定向 - 完成请求必须进行进一步操作
    • 301： 请求页面已经转移到新页面
    • 302：请求页面临时转移到新页面
    • 304：客户端有缓冲稳定并发出一个条件性请求，服务端告诉客户，之前的缓冲文档还可以使用
  • 4XX ：客户端错误 - 请求语法错误和请求无法实现
    • 400: 客户端语法错误
    • 401: 请求未经授权
    • 403: 页面禁止访问
    • 404: 请求资源找不到
  • 5XX ：服务端错误 - 服务端未能实现合法请求
    • 500: 服务端发生错误

http 持久链接 Connection:Keep-Alive 模式

只需要建立一条连接，然后在这条连接上 传输多个请求和响应。

http 管线化 在持久状态下完成 1.1 版本支持

只有 get 和 head 请求可以进行管线化，其他的不能

第一次创建连接不能启动管线化，因为你不知道服务器是否支持 http1.1 协议

由于服务端问题， 浏览器都默认未开启管线化
