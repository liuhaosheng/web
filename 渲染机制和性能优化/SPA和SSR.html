SPA 单页面应用

    优点
  • 页面之间的切换很快
  • 减少后端服务器的压力
  • 后端只提供API
    缺点
  • 首屏打开速度缓慢
  • 不利于 SEO

如何解决 SPA 首屏 加载速度慢的问题？

  • 将公用的 js 库通过 script 标签外部引入，减少 app.bundel 的大小
  • 配置路由，页面和组件使用懒加载方式引入
  • 加一个首屏 loading 图，提升用户体验度

SSR 服务器端渲染

    优点
  • 响应时间快
  • 有利于 SEO
    缺点
  • 占用更多的内存和CPU
  • 一些常用的浏览器API 无法正常使用 如 window alert
  • 开发调试有些麻烦，对于SPA 一些组件的生命周期管理会变的复杂
  • 某些因素导致 服务端渲染和 浏览器的结果不一样
